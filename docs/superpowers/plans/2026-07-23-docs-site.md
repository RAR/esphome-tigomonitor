# Documentation + Config Builder Site — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the standalone config-wizard Pages site with an Astro + Starlight documentation site that renders all project docs and folds the existing config builder in as one page.

**Architecture:** An Astro + Starlight project lives in `site/`, layered *around* the wizard's existing tested core (`boards.js`, `rules.js`, `yaml.js`, `wizard.js`, `lib/`, `test/`) which stays byte-for-byte unchanged. The 7 user docs move into Starlight's content collection; the wizard becomes an Astro component embedded in a Starlight page. GitHub Pages builds via `astro build` and deploys `site/dist`.

**Tech Stack:** Astro 5, `@astrojs/starlight`, `sharp`; Node 22; the wizard's vanilla ES-module core + `node --test` suite (unchanged).

## Global Constraints

- Deploy target is GitHub Pages at `https://rar.github.io/esphome-tigomonitor/` → Astro config **must** set `site: 'https://rar.github.io'` and `base: '/esphome-tigomonitor'`. No custom domain (no CNAME).
- `site/package.json` **must** declare `"type": "module"` — the `.mjs` tests import the `.js` core (`export`-syntax) as ESM; explicit type keeps that resolution correct.
- These files **must remain byte-for-byte unchanged**: `site/boards.js`, `site/rules.js`, `site/yaml.js`, `site/wizard.js`, `site/lib/apikey.mjs`, `site/lib/yaml-extract.mjs`, and all of `site/test/*.test.mjs`. The ported wizard reuses them by import, never by edit.
- The wizard's `node --test test/*.test.mjs` suite **must stay at 36 passing** after every task (it is the parity guarantee for the ported wizard).
- Every build check is `cd site && npm run build`; it must exit 0 (Starlight fails the build on broken internal doc links — this is the docs link-check).
- Do **not** touch `docs/superpowers/` (internal specs/plans) or `docs/images/` (kept for the root README). Only the 7 user-doc `.md` files are migrated and deleted.
- No JS framework (no React/Preact) in the wizard — it stays a plain client island.
- Commit messages end with the trailer: `Claude-Session: https://claude.ai/code/session_01CDQ9ccpnsu3YtiVkvPqZtr`
- Work happens on branch `feat/docs-site` (already created off `main`).
